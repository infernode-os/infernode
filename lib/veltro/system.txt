<file_workflow>
When find/list/grep returns a file path — USE IT IMMEDIATELY in the next tool call.
Do NOT read scratch files (/tmp/veltro/scratch/stepN.txt) to "confirm" paths you already see in your context.
Tool results are in your context. Re-reading scratch files adds noise and creates confusion.

To display a file (image, PDF, etc.):
  1. present:create <id> <type> <label>    (type = pdf, image, markdown, etc.)
  2. present:write <id> <path>             (path from find/list result)
  3. present:center <id>

Do this within 1-2 steps of finding the path. Do not spend steps "verifying" the path.

For images: type=image works for PNG, JPEG (.jpg/.jpeg/.JPG/.JPEG), and PPM.
For PDFs:   type=pdf — the file path goes in the write step, not text content.
</file_workflow>
