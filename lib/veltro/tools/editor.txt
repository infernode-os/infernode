editor - AI remote control for the Editor app (does NOT launch it).
This tool takes one string argument named "args". The first word of
"args" is the subcommand: read, write, append, save, open, goto, find,
replace, replaceall, addr, insert, delete, name, close, status.
To start the editor app first, use the launch tool with args = "editor".

Argument shapes — read carefully, paths are NOT a universal argument:
  write  takes literal <text> only — args = "write Hello world"
         (NOT a path; the text becomes the entire document body)
  append takes literal <text> only — args = "append more text"
  read   takes "body" or "addr" — args = "read body"
         (NOT a path; "body" is the document text)
  open   takes a <path>       — args = "open /usr/me/notes.txt"
  name   takes a <path>       — args = "name /usr/me/new.txt"
  goto   takes a <line>       — args = "goto 42"
  find   takes a <string>     — args = "find TODO"
  save, close, status, addr  take no arguments

To display a message to the user in the editor: args = "write Hello!"
Do NOT include any path or filename — write replaces the body with
exactly the literal text after "write ".

If the user says "launch the editor", use the launch tool, NOT this tool.

NOTE: This tool does NOT launch the editor. To start the editor,
use the launch tool: "launch editor". This tool sends commands to
an editor that is already visible in the presentation zone.

If the user says "launch the editor" or "open the editor", use
the launch tool, not this tool.

How it works: send subcommands as the args string to this tool. The
editor polls for commands every 500ms, so there is a brief delay
between sending a command and it taking effect. Internal file paths
under /tmp/veltro/editor/ are an implementation detail used by this
tool internally — never address them directly from your tool calls.
The only paths you ever pass to this tool are real files for the
"open" and "name" subcommands.

Commands (only work after "launch editor"):
  read [body]              Read document body text
  read addr                Read cursor position (line col)
  write <text>             Replace entire document body
  append <text>            Append text to body
  save                     Save current file
  open <path>              Open file in editor
  goto <line>              Move cursor to line number
  find <string>            Search for text in document
  replace <find> <repl>    Replace next occurrence
  replaceall <find> <repl> Replace all occurrences
  addr                     Get cursor position (line col)
  insert <ln> <col> <text> Insert text at position
  delete <sl> <sc> <el> <ec>  Delete text range
  name <path>              Set file path without loading
  close                    Close editor
  status                   Show document info (id, path, dirty)

Typical workflow:
  launch editor                  1. Start the editor app
  editor open /usr/me/file.b     2. Open a file
  editor read                    3. Read the document body
  editor goto 42                 4. Jump to line 42
  editor find TODO               5. Search for text
  editor write <new content>     6. Replace body
  editor save                    7. Save to disk
