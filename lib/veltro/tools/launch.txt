launch - Launch a GUI app in the presentation zone.
This tool takes one string argument named "args" containing the app
name (and any app-specific arguments). Confirmed apps include: shell,
charon, editor, fractals, keyring, man, view, lens, about.
Examples:
  args = "shell"                       (open the shell window)
  args = "editor"                      (open the editor)
  args = "charon https://example.com"  (open browser at URL)
  args = "list"                        (list available apps)
Use this tool whenever the user says "launch X", "open X", or "start X".
Do NOT call the X tool itself for launching — the X tool typically
controls an app that is already running.

Usage:
  Launch list                     — show available apps
  Launch <appname>                — launch app by name
  Launch charon <url>             — open Charon web browser at URL

Charon is the built-in web browser. To browse any website:
  Launch charon https://www.wikipedia.org
  Launch charon https://example.com
If Charon is already running, it is killed and relaunched at the new URL.

When an app has a matching control tool (editor, shell, charon, fractal, man),
launching the app automatically activates that tool. The app exposes its
state as files under /tmp/veltro/<appname>/ and the control tool reads
and writes those files on your behalf.

Other confirmed apps: clock, bounce, coffee, colors, date, editor,
about, view, rt, lens, man, shell, fractals, keyring, xenith.

Apps requiring Tk (not available): task, tetris, sh, ftree, deb.
