limbo - Author a complete, compilable Limbo source file.

  WHEN TO CALL
    Call ONLY when the user has explicitly asked you to write, author,
    create, generate, port, or "give me" Limbo code (.b files, Inferno
    modules, code that must compile on Inferno).

    DO NOT call on:
      - greetings ("hello", "hi", "hey")
      - questions about how something works
      - any message that does not contain an explicit authoring directive

  ARGS
    A single short natural-language sentence describing the program.
    Plain English only. Pass the user's request verbatim if it already
    describes the program; otherwise paraphrase in one sentence.

    args MUST NOT contain code tokens: { } ; "implement" "import"
    "include" "package main" "def main" "#include".

  RETURNS
    The Limbo source code in a single ```limbo fenced block.

  WHY THIS TOOL EXISTS
    The orchestrator's general-purpose training produces Go-style or
    Python-style code that the Inferno compiler rejects (verified 0/10
    compile-pass). This tool dispatches to a Limbo-trained model that
    produces compileable code (10/10 on the same scenarios). Even with
    a fallback to a non-specialised model, quality is more predictable
    when the request goes through this tool than when the orchestrator
    inlines its own attempt.

  EXAMPLES IN THIS DOCUMENT ARE DOCUMENTATION. Never copy an example's
  argument text as your own tool call. Construct args from the user's
  actual request.
