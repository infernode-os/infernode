# GoDis differential-testing harness

A harness that finds godis correctness bugs the cheap way: take a normal Go
program, run it through **both** `go run` and godis, and diff the output. It does
this across a whole corpus, under both emulator execution modes, and classifies
every result.

For each program it:

1. compiles the program with godis (the in-process `compiler` package),
2. runs the resulting Dis bytecode on the Inferno emulator under **`-c0`**
   (interpreter) and **`-c1`** (JIT),
3. runs the same program with **`go run`** to get the reference output,
4. diffs both emulator runs against Go and classifies the outcome.

## Why both `-c0` and `-c1`

The existing godis E2E suite (`compiler/compiler_test.go`) runs the interpreter
only. That silently masks JIT codegen bugs: a program can pass under `-c0` while
`-c1` miscompiles it. This harness diffs **both** modes against Go on every run,
so a `-c0`/`-c1` disagreement (`c0!=c1`) is its own, highest-signal class. Any
locked program that regresses in JIT-only fashion fails CI.

## Classification

| class          | meaning                                                            |
|----------------|-------------------------------------------------------------------|
| `match`        | both emu modes reproduced `go run` exactly                         |
| `c0!=c1`       | interpreter and JIT disagree with each other — JIT codegen bug     |
| `diverge`      | both emu modes agree but differ from `go run` — godis bug          |
| `crash`        | emulator faulted (`Broken:`, segfault, nil-deref) in both modes    |
| `compile-fail` | godis could not compile the program                                |
| `go-error`     | `go run` itself failed (no usable oracle) — never gated            |
| `skipped`      | excluded by a directive or the skip manifest                       |

Precedence: a `c0!=c1` disagreement always wins; otherwise output byte-identical
to `go run` is always `match` (a real fault can't reproduce Go's full output);
otherwise a fault marker escalates `diverge` to `crash`.

### The `println` oracle subtlety

Go's builtin `print`/`println` write to **stderr** and the spec leaves their
formatting (notably floats: `+5.000000e+000`) implementation-defined. godis
routes those builtins to the emulator's **stdout**. So the harness compares emu
stdout against `go run`'s **combined** stdout+stderr (captured through a single
writer to preserve interleaving). Programs that rely on builtin float formatting
are listed in the skip manifest, not treated as bugs.

## Corpus layout

```
tools/godis/
├── testdata/            existing single-file E2E programs (reused as corpus)
└── _corpus/             differential-test corpus (underscore => go tool ignores it)
    ├── seed_*.go        hand-written realistic programs (algorithms, generics, …)
    ├── gen_*.go         generated by cmd/gencorpus (combinatorial coverage)
    ├── locked.txt       programs validated to match go run — the CI regression gate
    └── skip.txt         programs with no faithful go oracle / nondeterministic
```

A program is either a single `.go` file or a directory of `.go` files
(multi-file / multi-package). Program names are paths relative to `tools/godis`
(e.g. `testdata/hello.go`, `_corpus/seed_quicksort.go`).

## Running it

From `tools/godis` (the emulator must be built — `../../build-linux-amd64.sh
headless` produces `emu/Linux/o.emu`):

```sh
# Diff the whole corpus, print a summary + the worklist of divergences.
go run ./cmd/difftest

# Just one directory, with JSON and Markdown reports.
go run ./cmd/difftest -json report.json -md report.md _corpus

# List what would be tested (and why anything is skipped).
go run ./cmd/difftest -list

# Run the regression gate locally (fails if a locked program drifts).
go run ./cmd/difftest -locked _corpus/locked.txt testdata _corpus
```

Useful flags: `-timeout` (per emu run; it hangs and is killed), `-jobs`
(parallelism), `-fail-on diverge,c0!=c1,crash` (exit nonzero on those classes),
`-skip <file>`, `-emu`/`-root` (override autodetection).

## Adding programs

- **Hand-written:** drop a deterministic `package main` into `_corpus/` named
  `seed_<name>.go`. Prefer `fmt` (stdout) over builtin `println`. Avoid
  goroutine/select scheduling, map iteration order, wall-clock time, and floats
  via builtins — or mark the file with `// difftest:skip <reason>`.
- **Generated:** extend `cmd/gencorpus` and regenerate:
  ```sh
  go run ./cmd/gencorpus     # rewrites _corpus/gen_*.go, leaves seed_*.go alone
  ```

## Promoting matches into the gate

When new programs (or newly-fixed ones) start matching `go run`, lock them in so
they can never silently regress:

```sh
go run ./cmd/difftest -locked _corpus/locked.txt -promote testdata _corpus
```

`-promote` only ever **adds** to `locked.txt`; a program that has regressed
stays listed so the gate keeps catching it.

## CI

The `godis` job in `.github/workflows/ci.yml`:

- runs `go test ./...`, which includes `difftest.TestE2ELocked` — the **hard
  gate**: every `locked.txt` program must still match under `-c0`, `-c1`, and
  `go run`;
- then runs `cmd/difftest` over the full corpus as a **non-blocking report**,
  publishing the diff to the job summary and uploading `difftest-report.json`,
  so new divergences are visible without breaking unrelated PRs.

The current ranked list of real divergences is in
[`../docs/DIFFTEST-FINDINGS.md`](../docs/DIFFTEST-FINDINGS.md).
